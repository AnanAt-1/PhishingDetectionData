Version 1.4 - 26, Mar, 2005

[26/03/2005] RedMSX 1.4
New version of RedMSX, what's new:
  • cas support! (only not zipped for now)
  • more mappers supported (R-Type now runs!)
  • Option to enable/disable sprites collisions
  • Option to enable/disable sprite rendering
  • And support to lightgun! The only format I know (sorry), which is used by Duck Hunt! by Karoshi Corp. So now, Duck Hunt is playable! I'm still looking for info on the ASCII lightgun, so I can make Dungeon Hunter work...
  • A little more of speed hacks when using directdraw render
With this, RedMSX is the first emulator to emulate lightgun! Happy Easter!


[09/18/2004] RedMSX 1.3
New version of RedMSX, what's new:
  • magnified sprites, for all supported screens!
  • fixed the bug on disk history for disk A and B
  • several Direct-X speed hacks
  • now drag and drop of zip files also works
  • mounting directories as dsk implemented
  • horizontal scroll on screen 4/5 (almost finished: now Space Manbow for MSX2+ and Sonyc both works!)
And since I got a lot of emails calling this emulator a "ripoff", saying that it's useless and worst stuff, there will be no more source relases. Live with it.


[12/12/2003] RedMSX 1.2
New version, now in sync with blueMSX 1.0.5 (but with different routines to load zip files), this version has other improvements: great speed up and multiple screenshot support (Press. "Print Screen" key).

** UPDATE **
I have made a mistake, and uploaded a wrong binary version. New one is online, and if you downloaded the old one, please delete it and use the new one.


[12/01/2003] RedMSX 1.1a
Inspite of some bad comments, a new version of RedMSX is born.
This one have support for zipped dsk files. The emulator will extract it to where RedMSX is, and then read the dsk from there, giving full read/write possibilities to it. Get the updated version on the download section.

** UPDATE ** A small bug has been corrected, the emulator wouldn't find the ROM or DSK file insite the zip if the extension was in uppercase. Now fixed.
