    Actualizare pentru a activa TLS 1.1 și TLS 1.2 ca protocoale sigure implicite în WinHTTP în Windows

    Applies ToWindows Server 2012 Datacenter Windows Server 2012 Standard Windows Server 2012 Essentials Windows Server 2012 Foundation Windows Server 2008 R2 Service Pack 1 Windows Server 2008 R2 Datacenter Windows Server 2008 R2 Enterprise Windows Server 2008 R2 Standard Windows Server 2008 R2 Foundation Windows Server 2008 R2 for Itanium-Based Systems Windows 7 Service Pack 1 Windows 7 Ultimate Windows 7 Enterprise Windows 7 Professional Windows 7 Home Premium Windows 7 Home Basic Windows 7 Starter

    Această actualizare oferă suport pentru Transport Layer Security (TLS) 1.1 și TLS 1.2 în Windows Server 2012, Windows 7 Service Pack 1 (SP1) și Windows Server 2008 R2 SP1.

    Despre această actualizare

    Aplicațiile și serviciile scrise utilizând conexiuni WinHTTP pentru conexiuni Secure Sockets Layer (SSL) care utilizează semnalizarea WINHTTP_OPTION_SECURE_PROTOCOLS nu pot utiliza protocoale TLS 1.1 sau TLS 1.2. Acest lucru se datorează faptului că definiția acestui semnalizator nu include aceste aplicații și servicii.

    Această actualizare adaugă suport pentru intrarea de registry DefaultSecureProtocols, care îi permite administratorului de sistem să specifice protocoalele SSL care trebuie utilizate atunci când WINHTTP_OPTION_SECURE_PROTOCOLS semnalizatorii.

    Acest lucru poate permite ca anumite aplicații care au fost construite să utilizeze semnalizarea implicită WinHTTP pentru a putea utiliza protocoalele TLS 1.2 sau TLS 1.1 mai noi, în mod nativ, fără a fi nevoie de actualizări ale aplicației.

    Acesta este cazul pentru unele aplicații Microsoft Office atunci când deschid documente dintr-o bibliotecă SharePoint sau dintr-un folder web, angajați IP-HTTPS pentru conectivitatea DirectAccess și alte aplicații utilizând tehnologii cum ar fi WebClient utilizând WebDav, WinRM și altele.

    Această actualizare necesită ca componenta Secure Channel (Schannel) din Windows 7 să fie configurată să acceptă TLS 1.1 și 1.2. Deoarece aceste versiuni de protocol nu sunt activate în mod implicit în Windows 7, trebuie să configurați setările de registry pentru a vă asigura că aplicațiile Office pot utiliza cu succes TLS 1.1 și 1.2.

    Această actualizare nu va modifica comportamentul aplicațiilor care setează manual protocoalele securizate în loc să treacă semnalizarea implicită.

    Cum se obține această actualizare

    Important Dacă instalați un pachet lingvistic după ce instalați această actualizare, trebuie să reinstalați această actualizare. Prin urmare, vă recomandăm să instalați toate pachetele lingvistice de care aveți nevoie înainte de a instala această actualizare. Pentru mai multe informații, consultați Adăugarea pachetelor lingvistice la Windows.

    Metoda 1: Windows Update

    Această actualizare este furnizată ca actualizare recomandată în Windows Update. Pentru mai multe informații despre cum să rulați Windows Update, consultați Cum se obține o actualizare prin Windows Update.

    Metoda 2: Catalogul Microsoft Update

    Pentru a obține pachetul stand-alone pentru această actualizare, accesați site-ul web Catalog Microsoft Update.

    Actualizați informațiile detaliate

    Cerinţe preliminare

    Pentru a aplica această actualizare, trebuie să instalați Service Pack 1 pentru Windows 7 sau Windows Server 2008 R2.

    Nu există nicio condiție necesară pentru a aplica această actualizare în Windows Server 2012.

    Informații despre registry

    Pentru a aplica această actualizare, trebuie adăugată subcheia de registry DefaultSecureProtocols.
    Notă Pentru a face acest lucru, puteți să adăugați manual subcheia de registry sau să instalați "Remedieresimplă " pentru a popula subcheia de registry.

    Cerință de repornire

    Poate fi nevoie să reporniți computerul după ce aplicați această actualizare.

    Actualizarea informațiilor de înlocuire

    Această actualizare nu înlocuiește o actualizare lansată anterior.

    Mai multe informații

    Domeniul cardurilor de plată (PCI) necesită TLS 1.1 sau TLS 1.2 pentru conformitate.

    Pentru mai multe informații despre semnalizarea WINHTTP_OPTION_SECURE_PROTOCOLS, consultați Semnalizări opțiune.

    Cum funcționează intrarea de registry DefaultSecureProtocols

    Important Această secțiune, metodă sau activitate conține pași care vă indică modalități de modificare a registry. Totuși, dacă modificați incorect sistemul registry, pot apărea probleme serioase. De aceea, asiguraţi-vă că urmaţi aceşti paşi cu atenţie. Pentru o protecție mai bună, înainte de a face modificări, realizați o copie de rezervă a sistemului registry. După aceea, sistemul registry poate fi restaurat dacă apare o problemă. Pentru mai multe informații despre cum să faceți backup și să restaurați registry, consultați Cum să faceți backup și să restaurați registry în Windows.

    Atunci când o aplicație specifică WINHTTP_OPTION_SECURE_PROTOCOLS, sistemul va căuta intrarea de registry DefaultSecureProtocols și, dacă există, va înlocui protocoalele implicite specificate de WINHTTP_OPTION_SECURE_PROTOCOLS cu protocoalele specificate în intrarea de registry. Dacă intrarea de registry nu este prezentă, WinHTTP va utiliza setările implicite ale sistemului de operare existent pentru Win WINHTTP_OPTION_SECURE_PROTOCOLS HTTP. Aceste winhttp defaults respectă regulile de prioritate existente și sunt anulate de protocoalele și protocoalele dezactivate SCHANNEL setate pentru fiecare aplicație de WinHttpSetOption.

    Notă Programul de instalare de remediere rapidă nu adaugă valoarea DefaultSecureProtocols. Administratorul trebuie să adauge manual intrarea după determinarea protocoalelor de înlocuire. Sau puteți instala "Remediere simplă"pentrua adăuga intrarea automat.

    Intrarea de registry DefaultSecureProtocols poate fi adăugată în calea următoare:

    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet Settings\WinHttp

    Pe computerele bazate pe x64, DefaultSecureProtocols trebuie de asemenea adăugat la calea Wow6432Node:

    HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\Windows\CurrentVersion\Internet Settings\WinHttp

    Valoarea de registry este un bitmap DWORD. Valoarea de utilizat este determinată prin adunarea valorilor corespunzătoare protocoalelor dorite.

    Valoarea DefaultSecureProtocols

    Protocol activat

    0x00000008

    Activați SSL 2.0 în mod implicit

    0x00000020

    Activați SSL 3.0 în mod implicit

    0x00000080

    Activarea implicită a TLS 1.0

    0x00000200

    Activarea implicită a TLS 1.1

    0x00000800

    Activarea implicită a TLS 1.2

    De exemplu:

    Administratorul dorește să suprascrie valorile implicite pentru WINHTTP_OPTION_SECURE_PROTOCOLS să specifice TLS 1.1 și TLS 1.2.

    Luați valoarea pentru TLS 1.1 (0x00000200) și valoarea pentru TLS 1.2 (0x00000800), apoi adunați-le în calculator (în modul programator) și valoarea de registry rezultată ar fi 0x00000A00.

    Remediere simplă

    Pentru a adăuga automat subcheia de registry DefaultSecureProtocols, faceți clic aici. În caseta de dialog Descărcare fișier, faceți clic pe Rulare sauDeschidere , apoi urmați pașii din expertul de remediere simplă.

    Note

    • Este posibil ca acest expert să fie disponibil numai în limba engleză. Cu toate acestea, remedierea automată funcționează și cu Windows în alte limbi.

    • Dacă nu vă a găsiți pe computerul care are problema, salvați soluția de remediere simplă pe o unitate flash sau pe un CD, apoi rulați-o pe computerul care are problema.

    Notă În plus față de subcheia de registry DefaultSecureProtocols, remedierea Easy adaugă, de asemenea, SecureProtocols în următoarea locație, pentru a ajuta la activarea TLS 1.1 și 1.2 pentru Internet Explorer.

    Intrarea de registry SecureProtocols cu 0xA80 pentru activarea TLS 1.1 și 1.2 va fi adăugată în următoarele căi:

    HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings

    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Internet Settings

    Activați TLS 1.1 și 1.2 pe Windows 7 la nivel de componentă SChannel

    Conform articolului Setări TLS-SSL,pentru ca TLS 1.1 și 1.2 să fie activat și negociat pe Windows 7, TREBUIE să creați intrarea "DisabledByDefault" în subcheia (Client) corespunzătoare și să o setați la "0". Aceste subchei nu vor fi create în registry, deoarece aceste protocoale sunt dezactivate în mod implicit.

    Creați subcheile necesare pentru TLS 1.1 și 1.2; creați valorile DWORD DisabledByDefault și setați-le la 0 în următoarele locații:

    Pentru TLS 1.1

    Locație de registry: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL\Protocols\TLS 1.1\Client
    Nume DWORD: DisabledByDefault
    Valoare DWORD: 0

    Pentru TLS 1.2

    Locație de registry: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL\Protocols\TLS 1.2\Client
    Nume DWORD: DisabledByDefault
    Valoare DWORD: 0
